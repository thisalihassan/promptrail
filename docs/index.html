Without Promptrail

You ask Cursor to refactor auth. It edits 4 files.

You ask it to add error handling. 6 more files.

Something breaks.

Git log shows one commit: "AI changes"
Cursor checkpoints: a flat list of file states.

You have no idea which prompt caused which change.

With Promptrail

$ promptrail timeline --files

● #0 cursor Refactor auth middleware  4 files
      src/middleware/auth.ts
      src/routes/login.ts
      src/routes/logout.ts
      src/utils/token.ts
● #1 cursor Add error handling  6 files
      src/middleware/error.ts
      src/routes/api.ts
      ...
○ #2 cursor Why is the build failing?  no changes
Features

Everything you need to trace AI changes

☰

Prompt Timeline

Every AI prompt listed with file change counts, model badges, and timestamps. promptrail timeline

⇄

Prompt-Scoped Diffs

Before/after diffs scoped to a single prompt, not the whole session. promptrail diff 3

↺

Cherry-Pick Rollback

Undo one prompt's changes without losing everything after it. promptrail rollback 5

🔎

Full-Text Search

Search across all prompts and AI responses with FTS5. promptrail search "auth"

💬

View AI Responses

Read the full AI reply for any prompt — text and tool calls. promptrail response 3

🛠

Zero Config

Reads existing session data on disk. No tagging, no buttons, no network calls. Just works.

How It Works

No interception. No configuration.

1

Use AI normally

Write prompts in Cursor or Claude Code as you always do. Nothing to set up.

2

Sessions are read automatically

Promptrail reads Cursor's SQLite database and Claude Code's JSONL session files. Everything stays local.

3

Browse, diff, rollback

Use the CLI, the Cursor extension sidebar, or the Claude Code plugin to trace every change.
